命令执行白名单（Layer 2）· 安全页设计稿

位置：安全（Security）页 → 新增卡片「命令执行白名单」，紧跟在「文件控制」卡片之后。
依赖：仅当「命令执行（shell_enabled）」已开启时本卡片才可配置；否则整卡置灰并提示先开启命令执行。
行为：开启后，每个 run_command 子命令的首个程序必须在白名单内（大小写不敏感、按程序名 basename 匹配），与 Layer 1 破坏性拦截叠加。空列表视为未启用，避免误锁死全部命令。

开启后，只有白名单内的程序可被远程执行（如 git、node、cmd）。与「破坏性操作拦截（Layer 1）」叠加，进一步缩小可执行面。需先开启命令执行。
关闭时：远程可在白名单目录内执行任意程序（仅受 Layer 1 保护）。开启后：未列入白名单的程序一律拦截，并向调用方返回「不在白名单内」。

匹配规则：大小写不敏感，按程序名 basename 比较（git 能匹配 /usr/bin/git、C:\Program Files\Git\bin\git.exe）。
添加时自动去重；删除最后一项不会自动关闭开关（空列表等价于「未启用」，所有命令照常放行，仅受 Layer 1 约束）。

与现状的衔接：「命令执行」开关（shell_enabled）已在安全概览内。本卡片仅新增 opt-in 的「可执行程序白名单」，复用同一 save_config 通道（patch 含 commandAllowlistEnabled / commandAllowlist）。后端校验（command_policy.rs）与配置字段（config.rs / commands.rs / types.ts）已在本任务落地并通过测试。
