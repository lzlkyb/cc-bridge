## A3 · 端口占用提示

### 现状（读真实代码）
- `Header.tsx:74-83` 状态胶囊只有三态：运行中（绿+脉冲）/ 已停止（灰）/ 连接中。无错误态。
- `SettingsTab.tsx:56-75` `handleSaveAndRestart` 里 `save_config` → 若 `restartRequired` 调 `restart_mcp_server`，整段在 `try/finally` 中但**无 catch 展示**，绑定失败（端口占用）会静默抛错、用户只看到按钮转圈后无反应。
- `StatusResponse` 无"启动错误"字段，后端绑定失败无法回传。

### 方案
1. **后端**
   - `types.ts`：`StatusResponse` 增 `startupError: string | null`（可选，向后兼容）。
   - `commands.rs` / `AppState`：新增 `startup_error: Mutex<Option<String>>`；`start_mcp_server` 绑定失败（`Address already in use` 等）捕获写入该字段；`get_status` 读出填入 `startupError`。
   - （实现时先读 `commands.rs::start_mcp_server` 确认错误捕获点，不臆测。）
2. **前端**
   - `Header.tsx` 状态胶囊增第四态「启动失败」：红边框 + `destructive` 色 + 文案 `启动失败：{startupError}`（如"端口 7823 被占用"），`title` 给完整原因。
   - `SettingsTab.tsx` `handleSaveAndRestart` 给 `restart_mcp_server` 单独 `try/catch`，失败 `toast(原因, "error")` 并 `onChanged()` 刷新（让 Header 红态也同步）。
3. **文件**：`desktop/src-tauri/src/commands.rs`、`http.rs`（若状态在此组装）、`desktop/src/lib/types.ts`、`Header.tsx`、`SettingsTab.tsx`
4. **风险**：低。仅新增可选字段 + 错误展示，不碰安全语义。

## B6 · 深色主题层次微调

### 现状（读真实代码）
`index.css:98-163` 暗色变量全定义：
- `--background: 240 6% 6%`（#09090B）
- `--card: 240 6% 10%`（#18181B）——比背景高 4% 明度
- `--muted: 240 4% 16%`、边框 `240 4% 19%`

层次存在但偏弱：暗色下卡片与背景仅靠 4% 明度差区分，长列表/多卡片时边界感不足；`.card-lift:hover` 暗色下阴影 `rgba(255,255,255,0.04)` 极弱，hover 反馈不明显。

### 方案
1. 暗色 `.dark` 块微调（**不改色相，只调明度/对比**）：
   - `--card` 由 `240 6% 10%` → `240 6% 12%`（卡片更"浮起"）
   - `--muted` 由 `240 4% 16%` → `240 4% 18%`（次级区更分明）
   - 边框 `240 4% 19%` → `240 4% 22%`（卡片描边更可见）
2. `.card-lift:hover` 暗色补充：加 `border-color: hsl(var(--border))` 提亮 + 微弱白边 `0 0 0 1px rgba(255,255,255,0.04)`，强化 hover 反馈。
3. **不引入新变量、不动浅色**，避免回归。
4. **文件**：仅 `desktop/src/index.css` 的 `.dark` 块 + `.card-lift` 规则。
5. **风险**：纯视觉，需你过目确认观感（用下方对照稿）。

`design/体验优化-B6-深色主题.html`（当前 vs 提议 对照）

## C8 · 配置导入 / 导出

### 现状（读真实代码）
- `SettingsTab.tsx` 配置散在 `NetworkGroup` / `AppGroup` / `AuditGroup` / `SettingsToggles`，无整体快照。
- 无导出/导入入口；跨机迁移只能逐项手填。

### 方案
1. **后端**（两命令，声明入 `commands.rs` + `http.rs` dispatch）
   - `export_config() -> String`：返回 `BridgeConfig` 美化 JSON（含白名单 roots + 5 个开关 + 网络配置 port 等）。
   - `import_config(json: String) -> Result<(), String>`：解析 → 逐项 `save_config` 应用；**导入的 `allowedRoots` 必须重走 `security/path.rs` 的 canonicalize + 祖先遍历校验**（规则7：安全模块不得放松，被拒时附白名单错误提示），杜绝导入绕过白名单。
2. **前端**（`SettingsTab.tsx`）
   - 新增「配置」卡片（放 `AppGroup` 之后），含：
     - **导出配置**：`export_config` → 前端生成 `.json` 下载（`Blob` + `<a download>`）或复制 JSON 到剪贴板（toast 提示）。
     - **导入配置**：`<input type="file" accept=".json">` 选文件 → 读文本 → `import_config` → 成功 `toast` + `onSaved()` 刷新全页；失败 toast 报错（含白名单拒绝原因）。
   - 复用现有 `Card` / `Button` / `Icon` / `useToast` 组件，不新增样式体系。
3. **文件**：`desktop/src-tauri/src/commands.rs`（+声明）、`http.rs` dispatch、`desktop/src/lib/types.ts`、`SettingsTab.tsx`
4. **风险**：中。导入必须复用安全校验路径，这是唯一红线；其余为常规 CRUD。
