# 还在用 scp 给远程 Claude Code 传文件？试试本地文件桥接 cc-bridge

> 一篇给「本地 Windows 写码 + 远程 Linux 跑 Claude Code」人群看的安利文。
> 适合首发：掘金 / 知乎专栏。配图位置已标注，用真实截图替换即可。

## 摘要

你在本地 Windows 写代码，Claude Code 跑在远程 Linux 服务器上——是不是每次都要 `scp` 把文件传上去，改完再 `scp` 拉回来？大文件慢、容易版本混乱、断网还卡死。

**cc-bridge** 就是来干掉这层 `scp` 的：装在你本机，远程的 Claude Code 就能像在本地一样，直接读、写、搜你的文件。而且它只有 **3.4MB**，安全默认拒绝一切越权访问，误删还能一键还原。

## 一、先说说 scp 有多烦

这不是我一个人的痛点。很多人的工作流是这样的：

- 本地 Windows 写代码，远程 Linux 上部署 Claude Code（毕竟显卡、环境都在服务器）。
- 想让 Claude Code 改个本地文件？先 `scp` 传上去 → 它改 → 再 `scp` 拉回来。
- 大文件慢、小文件烦，**最怕改到一半忘了哪边是最新的**，版本直接乱套。
- 试过 SSHFS 挂载？网络一抖挂载点就卡死，终端直接被阻塞，想骂人。

说白了：我们缺的不是一个更快的传输工具，而是**让远程 AI 直接"碰"到本地文件**的能力。

## 二、cc-bridge 是什么（一句话版）

**cc-bridge 是一个跑在你本机 Windows 上的小工具，把本地文件系统通过标准 MCP 协议，安全地"桥接"给远程的 Claude Code。**

打个比方：它像在你本机和远程 Claude Code 之间修了一条**专用内部管道**——Claude Code 不用知道文件在哪儿，直接说"读一下 A 文件""把 B 改成这样"，本机这边就替它把活干了。没有 scp，没有挂载，没有版本混乱。

## 三、它到底怎么干活的（白话版）

你双击启动 cc-bridge，它就在本机起一个轻量 HTTP 服务（监听内网端口 7823）。远程 Claude Code 用一条命令连上来：

```bash
claude mcp add --transport http cc-bridge http://<你本机IP>:7823/mcp \
  --header "Authorization: Bearer <令牌>"
```

连上之后，Claude Code 就能调用 **17 个文件操作工具**：读、写、精准改、删、移动、复制、搜内容、分析文件，甚至批量 `batch` 一次干好几件事、直接编辑 Jupyter 笔记本。

关键点：**两边共享同一份状态**。Claude Code 动了哪个文件，你本机面板上的审计日志实时就记下来了——不用瞎猜它改了啥。

## 四、安全吗？这是我最在意的

很多人第一反应是：让远程 AI 碰我本机文件，不怕出事？

cc-bridge 的设计就是**默认不信任**：

- **白名单**：只能访问你明确加进来的目录，其他一律拒绝（默认空名单 = 拒绝一切）。
- **令牌认证**：连接要带 Bearer Token，而且用了防时序攻击的常量时间比较。
- **审计日志**：每次操作都记时间、工具、参数、来源 IP、耗时，成功失败都记。
- **写前自动备份**：改文件 / 删文件之前先备份，手滑了能还原。
- **限流**：滑动窗口限流，防滥用。

> ⚠️ 一个实在的提醒：它默认走内网明文 HTTP，**只建议在 VPN 或受信任的内网里用**，别直接暴露到公网。

## 五、30 秒上手

1. 到 [Releases](https://github.com/lzlkyb/cc-bridge/releases) 下载安装包（国内走 Gitee 镜像更快），双击装好启动。
2. 打开「安全」页，把你的工作目录加进白名单。
3. 切到「连接」页，复制那行 `claude mcp add ...` 命令。
4. 粘到远程 Linux 终端执行——搞定，Claude Code 现在能直接碰你本机文件了。

## 六、为什么说它"小而美"

- **安装包 3.4MB**，二进制 14MB。对比早期 Node.js 方案 88MB→14MB 的瘦身，纯 Rust + Tauri 2 重写，不拖任何 sidecar。
- 原生桌面应用，开机自启、系统托盘常驻，点 X 是隐藏不是退出。
- 国内下载做了 Gitee 镜像优先 + 自动回退，GitHub 直连不稳也不怕。

对远程写码、又受够 scp 的人来说，这基本是「装完就回不去」的小工具。

## 七、适合谁 / 不适合谁

**适合**：本地 Windows + 远程 Linux 跑 Claude Code；受够文件来回传；在意安全和可审计。
**暂时不适合**：需要公网直连（目前只支持同内网/VPN）；非 Windows 本机（目前桌面端是 Windows）。
