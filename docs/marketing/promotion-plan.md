# cc-bridge 国内曝光推广方案

> 目标：让更多国内开发者知道并使用 cc-bridge（本地 MCP 文件桥接，让远程 Claude Code 直接读写本机文件）。
> 受众：国内开发者（前端 / 后端 / AI 编程爱好者 / 远程开发人群）。
> 调性：轻松好懂、说人话、用场景打动人。
> 所有对外文案统一引流到 GitHub 仓库：https://github.com/lzlkyb/cc-bridge

---

## 一、核心卖点（所有文案都要反复cue）

1. **替代 scp / SSHFS**：远程 Claude Code 像在本地一样直接读、写、搜你本机文件。
2. **安全默认拒绝**：路径白名单 + Bearer Token + 审计日志 + 写前自动备份，误删能还原。
3. **小而美**：纯 Rust + Tauri 2，安装包仅 **3.4MB**，二进制 14MB，无 Node.js 拖累。
4. **国内下载友好**：支持 Gitee 镜像优先 + 自动回退（GitHub / ghproxy），国内不卡。
5. **17 个文件操作工具**：含批量 `batch`、Jupyter 编辑 `notebook_edit`、GBK 编码自适应等。

### 各渠道要点
- **掘金**：带标签 `AI编程` `Claude Code` `效率工具` `Rust`；文内放架构图 + 截图；结尾放 GitHub 链接与「求 star」。
- **知乎**：搜索「Claude Code 怎么连本地文件」「远程开发 文件同步」类问题去回答，文末顺link；另发一篇专栏文章。
- **V2EX**：标题突出「替代 scp」；正文 200 字内讲清是什么 + 链接；附 GitHub Release 直链。
- **少数派**：偏产品体验，用「我怎么用 cc-bridge 远程写码」第一人称叙事；必须配真实界面截图。
- **HelloGitHub**：按投稿模板写 200 字项目简介 + 亮点 + 链接，每月一期，中了就有稳定曝光。
